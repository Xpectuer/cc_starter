# Requirements: Codex full_auto toggle via [s] key

## 1. Problem Statement

当前 TUI 中按 `s` 键仅对 Claude 后端有效（切换 `skip_permissions`）。Codex 后端的 `full_auto` 字段只能通过手动编辑 profiles.toml 或在 add form 中设置，无法在 TUI 主界面快速切换。这与 Claude 后端的体验不一致，增加了用户的操作成本。

## 2. Desired Outcome

在 Codex tab 下按 `s` 键，切换当前选中 profile 的 `full_auto` 字段，并通过 `toml_edit` 持久化到 profiles.toml。行为与 Claude 的 `skip_permissions` 切换完全对称。详情面板实时刷新显示新值，footer 提示文字根据当前后端动态切换。

## 3. Constraints
- **Tech stack**: Rust, ratatui, toml_edit（已有依赖，无需新增）
- **Hard boundaries**:
  - 必须使用 `toml_edit` 做 surgical edit，保留注释和格式
  - 不得引入新的外部依赖
  - 遵循现有的 `toggle_skip_permissions` 模式

## 4. Scope
### In Scope
- `config.rs`: 新增 `toggle_full_auto(profile_name, new_value)` 函数
- `main.rs`: `s` 键处理扩展 — Codex 后端调用 `toggle_full_auto`
- `ui.rs`: footer 快捷键提示根据 `active_backend` 动态切换（Claude: `s: toggle skip_permissions`，Codex: `s: toggle full_auto`）
- 对应的单元测试和集成测试

### Out of Scope
- `c` (continue) 键对 Codex 的支持（Codex 无 continue 概念）
- Codex 的 `skip_permissions` 支持（验证规则已禁止）
- UI 重构或布局变更

## 5. Acceptance Criteria
- [ ] `config::toggle_full_auto()` 可以切换指定 profile 的 `full_auto` 字段，并持久化到 TOML 文件
- [ ] 在 Codex tab 下按 `s`，当前 profile 的 `full_auto` 值取反，detail panel 立即刷新
- [ ] 在 Claude tab 下按 `s`，行为不变（仍切换 `skip_permissions`）
- [ ] footer 在 Claude tab 显示 `s: toggle skip_permissions`，在 Codex tab 显示 `s: toggle full_auto`
- [ ] `toggle_full_auto` 使用 `toml_edit` 保留注释和格式
- [ ] 所有现有测试继续通过
