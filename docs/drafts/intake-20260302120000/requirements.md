# Requirements: cct — Claude Code TUI Launcher

## 1. Problem Statement

The user maintains several shell wrapper functions (`kclaude`, `clauddy`, `minimax`,
`deepseek`, `ccr_claude`) in `~/claude_sub.sh`. Each function hardcodes a set of
environment variables (API base URL, auth token, model names, misc flags) and then
calls `claude $*`. This approach is fragile: adding a new provider means editing the
shell script, API keys are embedded in plain text with no central management, and
there is no discoverability (the user must remember all function names).

## 2. Desired Outcome

A single binary `cct` that, when run:
1. Opens a minimal terminal TUI showing all configured profiles.
2. Lets the user pick one with arrow keys and press Enter.
3. Injects the profile's env vars and extra CLI args, then `exec`s the `claude` binary.

Configuration lives in `~/.config/cc-tui/profiles.toml` — human-editable and version-
controllable. Pressing `e` in the TUI opens that file in `$EDITOR`.

## 3. Constraints

- **Tech stack**: Rust + ratatui (TUI), `toml` crate for config parsing.
- **Hard boundaries**:
  - Must `exec` (replace process) rather than spawn a subprocess, so claude inherits the terminal properly.
  - No network calls; no daemon; no background processes.
  - Config file path: `~/.config/cc-tui/profiles.toml` (XDG-compliant).

## 4. Scope

### In Scope

- Profile list TUI: arrow-key navigation, Enter to launch, `e` to edit config, `q`/`Ctrl-C` to quit.
- Per-profile fields:
  - `name` (string, required) — display name
  - `description` (string, optional) — shown in list as subtitle
  - `[env]` table — arbitrary key/value env vars injected before launch
  - `extra_args` (array of strings, optional) — extra CLI flags appended to `claude`
  - `skip_permissions` (bool, optional) — shorthand for `--dangerously-skip-permissions`
  - `model` (string, optional) — shorthand for `--model <value>`
- On launch: merge profile env vars into current process env, build final arg list, `exec` claude.
- Token masking: display `***` for any env value whose key contains `TOKEN`, `KEY`, or `SECRET`.
- On first run (no config file): auto-create `~/.config/cc-tui/profiles.toml` with a commented example profile.
- Import helper: `cct --import ~/claude_sub.sh` (stretch goal, not required for v1).

### Out of Scope

- In-app profile editor (forms) — use `$EDITOR` instead.
- GUI / web UI.
- Session management or history.
- Automatic secret encryption.

## 5. Acceptance Criteria

- [ ] `cargo build --release` produces a single static binary `cct`.
- [ ] `cct` renders a list of profiles from `~/.config/cc-tui/profiles.toml`.
- [ ] Arrow keys move selection; Enter launches claude with correct env+args.
- [ ] `e` key suspends TUI, opens `$EDITOR` on the TOML file, resumes on editor exit.
- [ ] `q` or `Ctrl-C` exits cleanly with code 0.
- [ ] Token/key values shown as `***` in the TUI.
- [ ] If no config file exists, a default one is created with an example profile.
- [ ] `extra_args`, `skip_permissions`, and `model` fields are correctly translated to
      CLI flags appended after `claude`.
