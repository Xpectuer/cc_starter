# Spec: cct install script

## Chosen Approach

**Approach A: Minimal single-function script** — a self-contained `install.sh` (~80-100 lines) with a `main()` wrapper, inline 3-retry loop with `tar -tzf` integrity verification, and no dependencies beyond `curl` + `tar`.

## Architecture

```
install.sh
├── Constants        REPO, INSTALL_DIR=~/.local/bin, MAX_RETRIES=3
├── Helpers
│   ├── err()        print to stderr + exit 1
│   ├── log()        print status messages
│   └── download()   curl + tar -tzf verify + retry loop (3 attempts, 2s sleep)
├── detect()         uname -s/-m → map to release target string
├── fetch_latest()   GitHub API /releases/latest → tag_name via grep/sed
├── install()        download tarball → extract to tmpdir → move to ~/.local/bin
└── main()           dep check → detect → fetch_latest → install → PATH hint
```

- `set -euo pipefail` for strict error handling
- `main()` wrapper called at EOF to prevent partial execution on network drop
- `trap` cleanup removes temp directory on exit
- No jq — parse JSON with grep/sed on stable fields (`tag_name`, `browser_download_url`)

## Target Mapping

| uname -s | uname -m | Release Target |
|----------|----------|----------------|
| Darwin | arm64 | aarch64-apple-darwin |
| Darwin | x86_64 | x86_64-apple-darwin |
| Linux | x86_64 | x86_64-unknown-linux-gnu |
| * | * | Error: unsupported platform |

## Data Flow

1. `detect()` — reads `uname -s` + `uname -m`, maps to TARGET string
2. `fetch_latest()` — `curl -fsSL` GitHub API, extracts VERSION tag
3. `download()` — constructs URL, downloads to tmpdir with retry:
   - Attempt 1..3: `curl -fSL` -> `tar -tzf` integrity check -> break on success
   - On failure: remove partial file, sleep 2s, retry
   - All 3 fail -> exit with error
4. `install()` — `tar -xzf` to tmpdir, `install -m 755` to `~/.local/bin/cct`
5. PATH hint — if `~/.local/bin` not in `$PATH`, print export line

## Error Handling

| Failure | Detection | Response |
|---------|-----------|----------|
| Unsupported OS/arch | case `*` | err "Unsupported platform: $OS $ARCH" |
| curl not found | `command -v curl` | err "curl is required" |
| tar not found | `command -v tar` | err "tar is required" |
| GitHub API failure | curl non-zero | err "Failed to fetch latest release" |
| Download/corrupt tarball | curl or tar -tzf fails | Retry up to 3 times, then err |
| ~/.local/bin not writable | install fails | err "Failed to install" |

## Verification Strategy

- `bash -n install.sh` — syntax check
- `shellcheck install.sh` — lint
- Manual test on macOS arm64, macOS x86_64, Linux x86_64
- Idempotency: re-run overwrites existing binary cleanly
