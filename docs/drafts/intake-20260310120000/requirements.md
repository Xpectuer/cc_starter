# Requirements: cct install script

## 1. Problem Statement

Users need a zero-dependency way to install `cct` without cloning the repo or building from source. The standard pattern `curl -fsSL <url> | bash` should download and install the latest release binary.

## 2. Desired Outcome

A single `install.sh` script in the repo root that:
1. Detects OS and architecture via `uname`
2. Maps to one of the 3 release targets (aarch64-apple-darwin, x86_64-apple-darwin, x86_64-unknown-linux-gnu)
3. Fetches the latest release tag from GitHub API
4. Downloads the matching `cct-{target}.tar.gz` asset
5. Extracts and installs the `cct` binary to `~/.local/bin/`
6. Prints a PATH hint if `~/.local/bin` is not in `$PATH`

Usage: `curl -fsSL https://raw.githubusercontent.com/zhengjy/cc_starter/master/install.sh | bash`

## 3. Constraints
- **Tech stack**: Pure POSIX shell (bash); requires only `curl` and `tar`
- **Hard boundaries**: No sudo, no wget fallback, no version pinning

## 4. Scope
### In Scope
- OS/arch detection (macOS arm64, macOS x86_64, Linux x86_64)
- GitHub API latest release lookup
- Download, extract, install to ~/.local/bin
- PATH check and hint message
- Error handling for unsupported platforms and download failures

### Out of Scope
- Version pinning (always latest)
- wget fallback
- ARM Linux support
- sudo / system-wide install
- Windows support
- Shell completion installation
- Config file bootstrapping

## 5. Acceptance Criteria
- [ ] `install.sh` exists at repo root
- [ ] Script detects OS and arch correctly on macOS (arm64, x86_64) and Linux (x86_64)
- [ ] Script exits with clear error on unsupported platform
- [ ] Script fetches latest release tag from GitHub API
- [ ] Script downloads correct `cct-{target}.tar.gz` asset
- [ ] Script extracts and places `cct` binary in `~/.local/bin/`
- [ ] Script creates `~/.local/bin/` if it doesn't exist
- [ ] Script prints PATH hint when `~/.local/bin` not in PATH
- [ ] Script prints success message with installed version
- [ ] Script fails gracefully if curl/tar not available
- [ ] Script is idempotent (re-running overwrites existing binary)
