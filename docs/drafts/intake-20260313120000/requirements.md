# Requirements: cct GLIBC Compatibility — musl Static Linking

## 1. Problem Statement

cct's Linux binary is built on GitHub Actions `ubuntu-latest` (glibc 2.35+). When deployed on Ubuntu 20.04 (glibc 2.31), the binary fails to start with multiple `GLIBC_2.3x not found` errors:

```
cct: /lib/x86_64-linux-gnu/libc.so.6: version `GLIBC_2.32' not found (required by cct)
cct: /lib/x86_64-linux-gnu/libc.so.6: version `GLIBC_2.33' not found (required by cct)
cct: /lib/x86_64-linux-gnu/libc.so.6: version `GLIBC_2.34' not found (required by cct)
cct: /lib/x86_64-linux-gnu/libc.so.6: version `GLIBC_2.39' not found (required by cct)
```

This prevents users on Ubuntu 20.04 and other older Linux distributions from using cct.

## 2. Desired Outcome

The release pipeline produces fully statically-linked Linux binaries (via musl) for both x86_64 and aarch64 architectures. These binaries run on any Linux distribution with kernel 2.6.32+ without glibc dependency.

## 3. Constraints
- **Tech stack**: Rust + GitHub Actions, musl target toolchain
- **Hard boundaries**: No glibc runtime dependency in released Linux binaries; minimum support target is Ubuntu 20.04 (glibc 2.31)

## 4. Scope
### In Scope
- Modify `.github/workflows/release.yml` to build with `x86_64-unknown-linux-musl` and `aarch64-unknown-linux-musl` targets
- Install musl toolchain and cross-compilation tools in CI
- Add CI smoke test to verify musl binaries work on Ubuntu 20.04

### Out of Scope
- install.sh changes (deferred to separate task)
- macOS build targets (unchanged)
- Windows support

## 5. Acceptance Criteria
- [ ] `file` command confirms Linux binaries are "statically linked"
- [ ] cct binary runs successfully (`cct --help`) in an Ubuntu 20.04 Docker container
- [ ] CI includes a smoke test step that validates musl binary functionality
