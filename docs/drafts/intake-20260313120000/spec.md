# Spec: cct musl Static Linking for Linux Binaries

## Chosen Approach

**方案 A: 原生 musl-tools + cross** — 在 GitHub Actions 中直接使用 `musl-tools` 和 `cross` 工具构建静态链接的 Linux 二进制。

### Rationale

- cct 是纯 Rust 项目，无 C 依赖（无 OpenSSL 等），musl 构建零障碍
- x86_64 直接用 `musl-tools` 即可，无需 Docker
- aarch64 用 `cross` 工具处理交叉编译
- 改动集中在 `release.yml` 一个文件，最小侵入

### Alternatives Considered

- **actions-rust-cross Action**: 封装好但引入第三方依赖，对 cct 来说不必要
- **BlackDex/rust-musl Docker**: 预编译 C 库的 Docker 镜像，对纯 Rust 项目过度设计

## Design Details

### 1. Matrix 变更

| Target | Runner | 构建工具 |
|--------|--------|----------|
| `aarch64-apple-darwin` | `macos-latest` | `cargo build` (不变) |
| `x86_64-apple-darwin` | `macos-15-intel` | `cargo build` (不变) |
| `x86_64-unknown-linux-musl` | `ubuntu-latest` | `cargo build --target` |
| `aarch64-unknown-linux-musl` | `ubuntu-latest` | `cross build --target` |

### 2. x86_64-musl 构建

1. `apt-get install -y musl-tools`
2. `rustup target add x86_64-unknown-linux-musl`
3. `cargo build --release --target x86_64-unknown-linux-musl`
4. 二进制路径: `target/x86_64-unknown-linux-musl/release/cct`

### 3. aarch64-musl 交叉编译

1. `cargo install cross --git https://github.com/cross-rs/cross`
2. `cross build --release --target aarch64-unknown-linux-musl`
3. 需要 Docker（cross 内部使用 Docker 进行交叉编译）

### 4. 条件构建逻辑

在 workflow 中通过 matrix 变量区分：
- Linux musl target → 安装 musl-tools / cross
- macOS target → 无额外步骤
- x86_64 Linux → `cargo build --target`
- aarch64 Linux → `cross build --target`

### 5. Package 步骤

二进制路径从 `target/release/cct` 统一为 `target/${{ matrix.target }}/release/cct`。
macOS 构建也需要加 `--target` 参数以保持路径一致。

### 6. 验证策略

**x86_64-musl:**
- `file` 命令确认 "statically linked"
- 直接运行 `cct --help`
- Docker `ubuntu:20.04` 烟雾测试

**aarch64-musl:**
- `file` 命令确认 "statically linked" + "aarch64" 架构
- 不运行烟雾测试（x86_64 runner 无法执行 aarch64 二进制，QEMU 暂不引入）
